Database Overview

risk-python queries four SQL Server databases. All are read-only from the perspective of these Python scripts — no INSERT/UPDATE/DELETE operations are performed by the automation scripts.

⚠️ Schema is reverse-engineered

Column types, nullability, and indexes are inferred from SQL query usage. The actual DDL must be confirmed against the SQL Server instance. Columns marked [INFERRED] have types estimated from context.

DatabaseServerPurposeScripts Using It
LHF_SYSTEMDefault instanceBloomberg MTM intraday dataBloomberg.sql
LHF_PERFORMANCEDefault instanceFund NAV performance historyFinal_VaR.sql, MF/PF/PVD_NAV.sql

LHF_SYSTEM — Bloomberg MTM Data

LHF_SYSTEM.DBO.LHF_BBG_DL_MTM_FX_EQ_MORNING

Stores Bloomberg Mark-to-Market (MTM) prices for FX and equity securities downloaded each morning. Used to produce a pivoted price matrix by date.

ColumnType [INFERRED]DescriptionNotes
MTMDATEDATEPrice dateUsed as row key in pivot; ORDER BY column
PRICECODENVARCHARBloomberg ticker / price codeBecomes column headers in pivot query
PX_LASTDECIMALLast priceAggregated with SUM in PIVOT (unique per PRICECODE per date)

Bloomberg.sql — Pivot Query Pattern

-- Dynamic pivot: each PRICECODE becomes a column
SELECT MTMDATE, [PRICECODE_A], [PRICECODE_B], ...
FROM LHF_BBG_DL_MTM_FX_EQ_MORNING
PIVOT (SUM(PX_LAST) FOR PRICECODE IN (...)) AS PivotTable
ORDER BY MTMDATE;

LHF_PERFORMANCE — Fund NAV

LHF_PERFORMANCE.dbo.ViewFundNavAll

A view (or table) containing the complete NAV history for all fund types. Core dataset for VaR and NAV reporting.

ColumnType [INFERRED]DescriptionNotes
FundCodeNVARCHARFund identifier (e.g. LHMM-A)Partition key in window functions
NAVDateDATENAV valuation dateFiltered to exclude weekends and holidays
NAVPerUnitDivDECIMALNAV per unit (after dividend adjustment)Used to compute DailyReturn via LAG()
FundTypeNVARCHARFund categoryValues: Mutual_Fund, Private_Fund, PVD

Derived Columns (computed in Final_VaR.sql)

ColumnFormulaDescription
Nav_previousLAG(NAVPerUnitDiv, 1) OVER (PARTITION BY FundCode ORDER BY NAVDate)Prior day NAV
DailyReturn(NAVPerUnitDiv / Nav_previous - 1) * 100Daily return %
RowNumROW_NUMBER() OVER (PARTITION BY FundCode ORDER BY NAVDate DESC)Used to select last N trading days for VaR window

VaR Query Logic (Final_VaR.sql)

The VaR calculation uses a parametric historical simulation approach over a configurable lookback window.

Parameters

ParameterPlaceholderDescription
@Date1###XXX###Reference date for VaR calculation (char 10)
@Numdate##YY#Number of trading days lookback window (e.g. 250)
@Confidence#X#YConfidence level (e.g. 0.95 or 0.99)

CTE Chain

RankedNAV  → computes DailyReturn per fund using LAG()
           → filters: weekends excluded, holidays excluded, null NAV excluded
CTEa       → adds ROW_NUMBER() DESC per fund, filters DailyReturn != 0, Mutual_Fund only
CTEb       → selects last @Numdate rows per fund (the lookback window)
[Final]    → applies percentile / confidence calculation for VaR
