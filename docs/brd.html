Executive Summary

Problem: The Risk team at LH Fund Management must process daily bond index data, NAV figures, VaR calculations, and performance reports. Previously this required manual downloads, Excel updates, and email composition — a time-consuming, error-prone process done before and after market hours each trading day.

Solution: risk-python automates the full morning and evening data pipeline: web scraping ThaiBMA bond indexes, triggering Excel VBA macros, and dispatching formatted email reports via Outlook — all with zero manual intervention.

Scope

In Scope

  • Morning: Automated login and data download from ThaiBMA (ibond.thaibma.or.th)
  • Morning: Copy downloaded files to network share
  • Evening: Trigger Excel VBA macros on power_automate_for_Afternoon.xlsm and Benchmark.xlsm
  • Evening: Send GPO performance update email via Outlook to risk and operations
  • Standalone: Send GPO benchmark email from Bloomberg Excel files
  • SQL: Queries for NAV, VaR, Holdings, Bloomberg MTM data (manual execution)

Out of Scope

  • Web dashboard or user interface
  • Real-time streaming data
  • Automated SQL query scheduling (currently manual)
  • Non-Windows operating systems
  • Any fund management or trading decision logic

Assumptions

  • Scripts run on a Windows machine with Office (Excel + Outlook) installed
  • Anaconda Python 3 is installed at C:\ProgramData\anaconda3\
  • Network shares (\\w2fsspho101.lhfund.net\FM-RI$\) are mounted and accessible
  • ThaiBMA credentials remain valid and website structure does not change
  • Chrome browser is installed for Selenium
