1Overview

risk-python is a Windows automation suite that runs the Risk team's daily morning and evening data workflows without manual intervention. It scrapes bond index data from ThaiBMA, runs Excel VBA macros, queries SQL Server for NAV/Holdings/VaR data, and sends email reports via Outlook — all triggered automatically by Windows Task Scheduler.

📦 Repository

https://github.com/clickzika/risk-python-new — branch main

⚠️ Windows Only

All scripts require Windows + Microsoft Office (Outlook + Excel) + Anaconda Python. They cannot run on Linux or macOS.

2Daily Schedule

All tasks run Monday–Friday. Tasks are skipped automatically on Thai public holidays.

  • 07:30
    Morning Part 1 — ThaiBMA login, download Morningstar Benchmark + MTMGov + CP + MTMCorp + ESG indexes. Copies files to network share and runs UpdateData macro.
    Runner: runners\run_morning.bat → scripts\morning\run_morning_part1.py
  • 07:30
    Morning Part 2 — Downloads ZRR, Short-term, Bond, Composite, Corp ZRR indexes. Runs UpdateData2 macro.
    Runner: runners\run_morning.bat → scripts\morning\run_morning_part2.py
  • 17:00
    Evening Workflow — Runs Create_Afternoon VBA macro, downloads evening ThaiBMA indexes, runs new_finish_set macro, sends GPO update email to risk + operation.
    Runner: runners\run_evening.bat → scripts\evening\run_evening.py
  • 17:15
    GPO Email — Standalone: reads Bloomberg GPO-FIXED and GPO-EQ Excel files, formats HTML table, sends via Outlook.
    Runner: runners\run_gpo_email.bat → scripts\evening\send_gpo_email.py
  • 17:30
    Dashboard Refresh — Regenerates docs\dashboard.html from latest run status files.
    Runner: runners\run_dashboard.bat → scripts\generate_dashboard.py
  • 18:00
    SQL Queries — Runs NAV (MF/PF/PVD), daily holdings, VaR, and Bloomberg MTM queries. Saves dated Excel files to \\w2fsspho101.lhfund.net\FM-RI$\risk\Amornsiri\SQL_Output\.
    Runner: runners\run_sql_queries.bat → scripts\run_sql_queries.py

3First-Time Setup

Perform these steps once on a new workstation before the scheduled tasks will work.

3.1 — Clone and install

git clone https://github.com/clickzika/risk-python-new.git
cd risk-python-new
pip install -r requirements.txt

3.2 — Create credentials file

copy scripts\.env.example scripts\.env
:: Then edit scripts\.env and fill in ThaiBMA username and password

3.3 — Set SQL Server hostname

Open scripts\config.py and replace:

DB_MAIN_SERVER = r'YOUR_MAIN_SQL_SERVER'

with the actual SQL Server hostname (e.g. r'192.168.1.10\SQLEXPRESS').

3.4 — Register scheduled tasks

Right-click runners\setup_scheduler.bat → Run as administrator.

This registers 5 tasks under LHFund\ in Windows Task Scheduler:

Task NameTime
LHFund\RiskMorningMon–Fri 07:30
LHFund\RiskEveningMon–Fri 17:00
LHFund\RiskGPOEmailMon–Fri 17:15
LHFund\RiskDashboardMon–Fri 17:30
LHFund\RiskSQLQueriesMon–Fri 18:00
⚠️ Workstation must stay on

The machine running these tasks must not be shut down or logged off before 18:00. Tasks run under the logged-in user session and require Outlook and Excel to be available.

5Troubleshooting

Where to look first

  1. Open docs\dashboard.html in a browser — the red card tells you which script failed.

  2. Open logs\{script_name}_YYYYMMDD.log — the last ERROR or CRITICAL line contains the full exception.

  3. Check your email — a failure alert is sent to risk@lhfund.co.th automatically.

Common issues

SymptomCauseFix
ThaiBMA login fails after 10 retries Wrong credentials or ThaiBMA site is down Check scripts\.env has correct user and pass. Try logging in manually at ibond.thaibma.or.th
FileNotFoundError on network share VPN not connected or share unavailable Connect to VPN / check \\w2fsspho101.lhfund.net is accessible in File Explorer
Edge WebDriver version mismatch Edge browser was updated but ChromeDriver is stale Run pip install --upgrade webdriver-manager — it auto-downloads the matching driver
Excel macro error / COM exception Excel file is open or locked by another user Close the relevant .xlsm file, then re-run manually
Outlook COMException Outlook is not running or not the default mail client Open Outlook manually and ensure it is signed in, then re-run
SQL query fails with login error Windows auth — running user has no DB access Ask DBA to grant db_datareader to the task scheduler service account
Script skips on a non-holiday Holiday DB query failed and defaulted to "not holiday"…or date is wrong in holiday table Check FIN_REG_LHF.dbo.holiday table for the date. Check connectivity to 192.168.102.7\DB2008
Task Scheduler shows "Last Run Result: 0x1" Script exited with non-zero code (Python exception) Open the corresponding log file in logs\ and look for CRITICAL
🚨 If morning data is missing

Run schtasks /run /tn "LHFund\RiskMorning" from an Administrator command prompt to trigger the morning job manually. Files will be copied to the network share as normal.

6Monitoring Dashboard

Open docs\dashboard.html in any browser. It shows a card per script with last run status (✅ SUCCESS / ❌ FAILED / ⏭ SKIPPED), timestamp, and detail (last file copied or error message). The page auto-refreshes every 5 minutes.

The dashboard is regenerated daily at 17:30 by LHFund\RiskDashboard. To regenerate manually: 

python scripts\generate_dashboard.py
📁 Status files

Raw status data lives in memory\status\{script}.json. These are plain JSON — you can inspect them directly if the dashboard is not loading.

7Running Manually

To trigger any task immediately without waiting for the schedule:

Via Task Scheduler (recommended)

schtasks /run /tn "LHFund\RiskMorning"
schtasks /run /tn "LHFund\RiskEvening"
schtasks /run /tn "LHFund\RiskGPOEmail"
schtasks /run /tn "LHFund\RiskSQLQueries"
schtasks /run /tn "LHFund\RiskDashboard"

Via command line (project root)

python scripts\morning\run_morning_part1.py
python scripts\morning\run_morning_part2.py
python scripts\evening\run_evening.py
python scripts\evening\send_gpo_email.py
python scripts\run_sql_queries.py
python scripts\generate_dashboard.py

Via bat files

runners\run_morning.bat
runners\run_evening.bat
runners\run_gpo_email.bat
runners\run_sql_queries.bat
runners\run_dashboard.bat
⚠️ Run from project root

Always run scripts from the project root directory (D:\Work\risk-python\), not from inside the scripts\ subfolder. The bat files handle this automatically.

10Known Limitations

#LimitationImpactWorkaround
1 Windows-only — requires Office + Anaconda Cannot run on a server without a desktop session Use a dedicated Windows workstation; ensure it is not locked during run hours
2 No automated tests Code changes are not regression-tested Manually smoke-test after any code change; CI only checks style (flake8)
3 ThaiBMA XPaths may break if website is redesigned Downloads fail silently (logged as WARNING) Inspect the ThaiBMA page manually and update XPaths in the relevant script
4 SQL date range is last 30 calendar days (hardcoded) Long holiday gaps may miss some dates Adjust _date_range_30d() in scripts\run_sql_queries.py
5 Outlook must be open and signed in Email alerts and GPO emails fail if Outlook is closed Configure Outlook to start with Windows login
